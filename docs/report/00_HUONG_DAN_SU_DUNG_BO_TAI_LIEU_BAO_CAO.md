# Hướng Dẫn Sử Dụng Bộ Tài Liệu Báo Cáo

## 1. Mục đích

Bộ tài liệu này giúp chuẩn bị nhanh cho:

- Báo cáo đồ án Cloud Computing.
- Vấn đáp kỹ thuật.
- Thuyết trình demo hệ thống.
- Bàn giao dự án cho thành viên mới.

## 2. Cấu trúc bộ tài liệu

1. `01_TONG_QUAN_VA_MUC_TIEU_DO_AN.md`
2. `02_KIEN_TRUC_CLOUD_VA_HA_TANG_HE_THONG.md`
3. `03_THIET_KE_KY_THUAT_BACKEND_FRONTEND_DU_LIEU.md`
4. `04_TRIEN_KHAI_VAN_HANH_CICD_VA_MO_RONG.md`

## 3. Cách dùng để viết báo cáo

- Chương mở đầu: dùng file `01_...`.
- Chương kiến trúc cloud và hạ tầng: dùng file `02_...`.
- Chương thiết kế kỹ thuật hệ thống: dùng file `03_...`.
- Chương triển khai, vận hành, CI/CD và đánh giá hiện trạng: dùng file `04_...`.

## 4. Cách dùng để thuyết trình 10-15 phút

1. Nêu bài toán và mục tiêu kỹ thuật.
2. Trình bày kiến trúc cloud tổng thể bằng sơ đồ.
3. Đi vào luồng hoạt động chính: API, realtime, CI/CD.
4. Nêu hiện trạng production và nút thắt.
5. Đề xuất lộ trình nâng cấp cloud-native.

## 5. Lưu ý khi vấn đáp

- Nhấn mạnh phần cloud được áp dụng ở lớp hạ tầng và lớp vận hành.
- Phân biệt rõ cái đã chạy ổn định và cái đang là rủi ro.
- Luôn có số liệu hoặc trạng thái thực tế để chứng minh.
